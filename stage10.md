# Stage 10: AI 추천 파이프라인 안정화 및 UI 롤백 & 구동 스크립트 추가

## 📌 주요 작업 내역

### 1. 메인 페이지 UI 롤백 및 디자인 다듬기
* **메인페이지 롤백**: 사용자의 요청에 따라 다른 페이지 디자인은 유지한 채, 메인 페이지(`index.html`, `index.css`)를 기존의 깔끔한 버전(커밋: `efc3c7c50593`)으로 롤백하여 귀여운 Tossface 이모지 기반 디자인을 원상복구했습니다.
* **챗봇 위젯 추가**: 메인 페이지 홈 화면 하단에도 챗봇 코드를 주입하여, 메인 화면 진입 시점부터 바로 AI 챗봇과 대화가 가능하도록 적용했습니다.
* **헤더(Navbar) 개선**: 스크롤 시 화면 상단 네비게이션 바에 강제로 생기던 회색/어두운 음영바 효과를 제거하여, 전체적으로 투명하고 깨끗한 뷰를 유지하도록 `global.css`를 수정했습니다.

### 2. AI 추천 및 RPA 크롤링 기능 안정화
* **AI 추천 기능 (JSON 파싱 에러 수정)**: 추천받기 클릭 시 파이썬 AI(`predict.py`)가 백그라운드에서 동작할 때, TensorFlow 및 패키지 경고(Warning) 로그가 자바의 파서로 흘러들어가면서 JSON 구조가 깨져 응답을 받지 못하던 문제를 발견했습니다. 파이썬 스크립트에서 불필요한 경고를 강제 억제(`os.environ['TF_CPP_MIN_LOG_LEVEL'] = '3'`)하여 순수 JSON만 반환되도록 완벽히 수정했습니다.
* **Conda 가상환경 정상 연동**: 자바 내부의 `AiRecommendService` 및 `DiscountCrawlerService`가 시스템 파이썬이 아닌 `C:\Users\pc06-00\anaconda3\envs\myenv\python.exe` 가상환경 경로를 1순위로 정확히 찾아가도록 확정 점검했습니다.
* **장보기 버튼 (RPA 동작)**: 이마트 등 사이트 UI 개편 시 추출 목록이 0개가 되더라도, 즉각적으로 메인 홈으로 우회하여 수집을 재시도하는 예외처리 및 지연 로직이 완벽하게 동작하는 것을 확인했습니다.

### 3. 클라우드 배포 환경 점검 가이드
* **Render 배포 500 에러 해결책 도출**: Render에서 "Failed to obtain JDBC Connection" 에러가 발생하는 이유는 코딩 문제가 아니라 TiDB Cloud의 보안 정책 때문입니다. TiDB 콘솔의 **Security -> IP Access List** 설정에 `0.0.0.0/0` (Allow All)을 추가해야 유동 IP를 쓰는 클라우드(Render)가 데이터베이스에 접근할 수 있음을 분석하여 가이드했습니다.

## 🚀 동작 및 실행 방법 (VSCode 없이 탐색기에서)

VSCode 등의 에디터를 열 필요 없이, 윈도우 파일 탐색기에서 바로 서버를 실행하고 테스트할 수 있습니다.

### 실행 방법
1. 윈도우 파일 탐색기에서 프로젝트 루트 폴더(`D:\wi\lab\recipekr\`)를 엽니다.
2. 새로 만들어진 **`start-recipekr.bat`** 파일을 찾아 더블클릭합니다.
3. 자동으로 검은색 명령 프롬프트 터미널 창이 열리며, 로컬 데이터베이스(`tidb`) 프로필로 Spring Boot 서버가 구동됩니다.
4. 로딩이 완료되면(터미널에 멈춘 듯 대기 상태가 되면), 웹 브라우저를 켜고 `http://localhost:8080` 에 접속합니다.
5. **종료**하실 때는 열려있는 검은색 명령 프롬프트 터미널 창을 X 버튼을 눌러 닫으시면 됩니다.

### 주요 기능 테스트
* **추천받기**: 상단 메뉴의 "레시피 추천"으로 들어가 냉장고 문을 연 뒤, 재료를 클릭해 [추천 받기] 버튼을 누르면 AI 스크립트가 실행되어 레시피가 생성됩니다. (약 5~10초 대기)
* **장보기(RPA)**: 동일한 화면에서 [장보기] 버튼을 누르면 파이썬 Playwright 브라우저가 보이지 않는 환경에서 이마트/롯데마트에 접속해 할인 정보를 긁어옵니다. (약 15~25초 대기 후 화면 자동 새로고침)
